01 What Chance Is

A canonical engine for resolving markets.

Chance is an OP-based rollup with one core primitive: a chain-level system for resolving prediction markets.

Builders stake collateral to create markets. Resolvers stake to publish outcomes. Stakers earn a share of fees. The chain collects protocol revenue from all prediction activity.

Chance ships with its own DeFi stack to support prediction flow.

02 The Problem

Why prediction markets break.

  • Resolution is unclear or centralised
  • Anyone can publish junk markets
  • Each project builds its own oracle, dispute flow, and settlement logic

Liquidity scatters. Users lose trust. Builders waste time.

03 The Chance Model

Resolution at the chain level.

PredictionHub

A chain-level system for resolving markets. Stores outcomes. Handles disputes. Routes protocol fees.

  • Single canonical contract: PredictionHub.sol
  • Markets register with resolver modules via standard interface
  • Outcomes stored as final, immutable events
  • Automated resolution for price feeds via Pyth/Redstone adapters
  • Template resolvers for sports, crypto, macro, news
  • Optimistic dispute model with bonds and escalation paths
  • Canonical indexing surface for all builders
  • FeeRouter triggered on resolution
  • Everything on-chain and verifiable

Stake-to-Create

Builders post collateral to create markets or categories. Spam becomes costly. Misbehaviour is slashed.

  • MarketStake.sol manages builder stakes
  • Stakes can be ETH, stETH, or LSTs
  • Single stake can back many markets
  • Slashing rules:
    • False outcomes
    • Abandoned markets
    • Broken resolvers
    • Misconfigured oracle parameters
  • Stake returned on correct resolution and clean history
  • Slashed stake auto-routed to stakers and treasury
  • Creates a real cost to low-quality markets

Protocol Fees

Every resolved market pays the chain. Fees flow to stakers, resolvers, builders, and treasury.

  • FeeRouter.sol directs resolution fees
  • Triggered on PredictionHub.finalizeOutcome()
  • Fee split configurable by governance (with timelocks)
  • Default split:
    • StakerVault (ETH yield for stakers)
    • Resolver operator reward
    • Builder rebate
    • Treasury for grants + infra
  • Fee sources:
    • AMM trading
    • Market creation
    • Resolution calls
    • Liquidations (later)
    • Slashed stakes
  • Fully on-chain. No off-chain accounting.

Resolvers & Automation

Pluggable resolver modules with keeper infrastructure for automated market settlement.

  • Pluggable resolver modules implementing IResolver
  • Operators stake to register resolver modules
  • Keepers run resolution bots
  • Automated price-based resolution via Pyth / Redstone
  • Batch settlement support for high-volume events
  • All resolver logic stored via ResolverRegistry.sol

Canonical DeFi Stack

Native financial primitives designed around prediction market flows and liquidity.

  • Binary & scalar AMMs built on CPMM/LMSR
  • Auto-payout integrated with PredictionHub
  • DEX base layer (v2/v3)
  • LP vaults for prediction markets
  • Keeper automation for liquidity maintenance
  • Indexer schema for all market and AMM events
  • Extensible SDK for custom market creators

05 Resolution System

Three resolution paths. One source of truth.

Resolvers post stake. Good resolvers earn fees. Poor resolvers are slashed.

Automated

Price-based markets resolve through Pyth, Redstone, or adapters. Keepers trigger settlement.

Template

Sports, politics, macro, and news markets use shared resolvers. Operators handle updates.

Quirky

Optimistic proposals with dispute windows. Escalation only when challenged.
